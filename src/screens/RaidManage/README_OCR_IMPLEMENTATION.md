## 🎮 스크린샷 OCR 혈맹원 감지 기능 - 구현 완료

### ✅ 완료된 기능

#### 1. 프론트엔드 구현 (100% 완료)

**생성된 파일:**

1. **`src/utils/ocrUtils.ts`** - OCR 유틸리티 함수
   - `calculateSimilarity()`: 텍스트 유사도 계산 (Levenshtein 거리 기반)
   - `normalizeOCRText()`: OCR 텍스트 정규화
   - `matchMembersWithOCR()`: 혈맹원 자동 매칭
   - `fileToBase64()`: 파일 변환
   - `preprocessImage()`: 이미지 전처리

2. **`src/components/modals/ScreenshotOCR.tsx`** - OCR 모달 컴포넌트
   - 파일 업로드 (클릭/드래그 드롭)
   - 이미지 미리보기
   - OCR 처리 및 결과 표시
   - 혈맹원 선택 UI
   - 대안 목록 제시
   - 수작업 수정 기능

3. **`src/screens/RaidManage/RaidManage.tsx`** - 통합
   - ScreenshotOCR 임포트
   - OCR 모달 상태 관리
   - `handleOCRSelect()`: OCR 결과 처리
   - 드래프트 행에 자동 추가
   - "📸 스크린샷 OCR" 버튼 추가

#### 2. 주요 기능

✅ **스크린샷 업로드**

- 파일 클릭 선택
- 드래그 드롭
- 미리보기 표시
- 파일 크기 제한 (5MB)

✅ **자동 매칭**

- 유사도 기반 혈맹원 감지
- 신뢰도 60% 이상 자동 선택
- 낮은 신뢰도는 미분류

✅ **수작업 수정**

- 대안 목록 제시
- 클릭으로 다른 혈맹원 선택
- "미분류" 옵션 (제외)
- 실시간 UI 업데이트

✅ **드래프트 통합**

- 선택된 혈맹원을 드래프트 행으로 추가
- 루팅자 필드 자동 채우기
- 기존 드래프트와 호환

#### 3. UX 개선사항

✨ **직관적 UI**

- 아이콘 사용 (📸, ✨, ✓, ✕, ▶)

- 신뢰도 퍼센트 표시
- 진행 상황 카운터 (n명 매칭, m명 미분류)

✨ **접근성**

- 모바일 대응 (반응형 디자인)
- 스크롤 가능한 결과 목록

### 🔄 처리 로직

**1. 텍스트 추출**

```
이미지 파일
  ↓
[백엔드 OCR]
  ↓
추출된 텍스트 (라인 단위)
```

**2. 자동 매칭**

```
추출된 이름 (e.g., "캐릭터이름1")
  ↓
혈맹원 리스트와 비교
  ├─ 정확히 일치? → 100% 신뢰도
  ├─ 부분 일치? → 유사도 계산
  └─ 첫 글자 일치? → 낮은 신뢰도
  ↓
신뢰도 >= 60% → 자동 선택
신뢰도 < 60% → 미분류 (대안 제시)
혈원에 없음 → 미분류 (제외 옵션)
```

**3. 결과 제시**

```
각 감지된 이름
  ├─ 자동 매칭 (녹색)
  ├─ 미분류 (노란색)
  └─ 대안 3개 제시
```

### 📊 신뢰도 기준

| 신뢰도 | 상태      | 처리                 |
| ------ | --------- | -------------------- |
| 100%   | 정확 일치 | 자동 선택 ✓          |
| 80-99% | 매우 유사 | 자동 선택 ✓          |
| 60-79% | 유사      | 자동 선택 ✓          |
| 40-59% | 약간 유사 | 미분류 ⚠ (대안 제시) |
| 1-39%  | 다름      | 미분류 ⚠ (대안 제시) |
| 0%     | 없음      | 미분류 ⚠ (제외 옵션) |
